Types API Reference

This page documents the core data types in Siena.jl.

Node Sets

NodeSet

Siena.NodeSet — Type
NodeSet

Represents a set of actors/nodes in the network.

Fields

  • n::Int: Number of nodes
  • names::Vector{String}: Optional node names
  • id::Symbol: Identifier for the node set
source

Data Container

SienaData

Siena.SienaData — Type
SienaData

Container for all data needed for SAOM estimation.

Fields

  • nodesets::Dict{Symbol, NodeSet}: Named node sets
  • dependents::Dict{Symbol, AbstractDependent}: Dependent variables
  • covariates::Dict{Symbol, AbstractCovariate}: Covariates
  • composition_change::Union{CompositionChange, Nothing}: Composition changes
  • n_waves::Int: Number of observation waves
source

Dependent Variables

AbstractDependent

Abstract type for dependent variables in SAOM.

DependentNetwork

Siena.DependentNetwork — Type
DependentNetwork

A dependent network variable observed at multiple time points.

Fields

  • name::Symbol: Variable name
  • networks::Vector{Matrix{Int}}: Network adjacency matrices at each observation
  • type::Symbol: Network type (:onemode, :twomode, :bipartite)
  • directed::Bool: Whether the network is directed
  • allow_self_loops::Bool: Whether self-loops are allowed
  • nodeset1::Symbol: ID of the first node set
  • nodeset2::Union{Symbol, Nothing}: ID of the second node set (for bipartite)
source

DependentBehavior

Siena.DependentBehavior — Type
DependentBehavior

A dependent behavioral variable observed at multiple time points.

Fields

  • name::Symbol: Variable name
  • values::Vector{Vector{Int}}: Behavior values at each observation
  • min_val::Int: Minimum allowed value
  • max_val::Int: Maximum allowed value
  • nodeset::Symbol: ID of the node set
source

ConstantCovariate

Siena.ConstantCovariate — Type
ConstantCovariate

A covariate that is constant across all waves.

Network State

The NetworkState type maintains the current state of networks and behaviors during simulation. It tracks adjacency matrices and behavior vectors as they evolve through mini-steps.

NetworkState

Siena.NetworkState — Type
NetworkState

Mutable state of networks and behaviors during simulation.

Fields

  • networks::Dict{Symbol, Matrix{Int}}: Current network states
  • behaviors::Dict{Symbol, Vector{Int}}: Current behavior states
  • time::Float64: Current simulation time within period
source

Algorithm Configuration

SienaAlgorithm

Siena.SienaAlgorithm — Type
SienaAlgorithm

Configuration for the SAOM estimation algorithm.

Fields

  • n_subphases::Int: Number of subphases in phase 2
  • phase1_iterations::Int: Iterations in phase 1
  • phase3_iterations::Int: Iterations in phase 3
  • initial_gain::Float64: Initial gain parameter (a)
  • min_gain::Float64: Minimum gain parameter
  • max_iterations::Int: Maximum total iterations
  • convergence_threshold::Float64: Convergence criterion (t-ratio threshold)
  • seed::Union{Int, Nothing}: Random seed
  • model_type::Symbol: Model type (:standard, :behavioronly, :networkonly)
  • conditional::Bool: Conditional estimation
  • n_simulations::Int: Number of simulations per iteration
  • parallel::Bool: Use parallel processing
  • verbose::Bool: Print progress
source

Effects Management Types

EffectEntry

Siena.EffectEntry — Type
EffectEntry

An entry in the effects table, combining an effect with its inclusion status and parameters.

Fields

  • effect::AbstractEffect: The effect itself
  • name::String: Display name
  • shortname::String: Short name for output
  • include::Bool: Whether effect is included in model
  • fix::Bool: Whether parameter is fixed
  • test::Bool: Whether to perform score test
  • initial_value::Float64: Initial parameter value
  • parameter::Int: Parameter group (for rate effects)
source
